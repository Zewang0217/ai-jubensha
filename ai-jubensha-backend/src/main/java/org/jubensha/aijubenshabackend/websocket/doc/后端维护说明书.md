# WebSocket 后端维护说明书

## 模块结构

```
websocket/
├── config/
│   └── WebSocketConfig.java              # WebSocket配置
├── controller/
│   └── WebSocketController.java          # 消息控制器
├── interceptor/
│   ├── GameHandshakeInterceptor.java     # 握手拦截器（分配gamePlayerId）
│   └── GameChannelInterceptor.java       # 通道拦截器（注册session）
├── listener/
│   └── WebSocketEventListener.java       # 连接断开监听
├── message/
│   └── WebSocketMessage.java             # 消息实体
├── service/
│   ├── PlayerSessionManager.java         # Session管理
│   ├── WebSocketService.java             # 服务接口
│   └── WebSocketServiceImpl.java         # 服务实现
└── doc/
    ├── 前端接入说明书.md
    └── 后端维护说明书.md
```

## 核心流程

### 1. 连接建立流程

```
前端连接 /ws?gameId=123
    ↓
GameHandshakeInterceptor.beforeHandshake()
    - 解析 gameId
    - 查询真人玩家列表 (GamePlayerService.getRealPlayersByGameId())
    - 按顺序分配 gamePlayerId
    - 存储到 session attributes
    ↓
STOMP CONNECT
    ↓
GameChannelInterceptor.preSend()
    - 从 attributes 获取 gamePlayerId
    - 注册到 PlayerSessionManager
    ↓
连接建立成功
```

### 2. 消息发送流程

```
业务代码调用 WebSocketService
    ↓
根据 gamePlayerId 从 PlayerSessionManager 获取 session
    ↓
使用 SimpMessagingTemplate 发送消息
```

## 关键类说明

### GameHandshakeInterceptor

- **作用**：WebSocket 握手时分配 gamePlayerId
- **核心逻辑**：
    - 从 URL 参数获取 gameId
    - 查询该游戏的真人玩家列表
    - 使用 AtomicInteger 按连接顺序分配
    - 无剩余玩家时拒绝连接

### GameChannelInterceptor

- **作用**：STOMP CONNECT 时注册 session
- **核心逻辑**：
    - 拦截 CONNECT 命令
    - 从 session attributes 读取 gamePlayerId
    - 调用 PlayerSessionManager.register()

### PlayerSessionManager

- **作用**：管理 sessionId 与 gamePlayerId 的映射
- **数据结构**：
    - `Map<sessionId, gamePlayerId>`
    - `Map<gamePlayerId, sessionId>`
- **方法**：
    - `register(sessionId, gamePlayerId)` - 注册映射
    - `getGamePlayerId(sessionId)` - 根据 session 获取玩家ID
    - `getSessionId(gamePlayerId)` - 根据玩家ID获取 session
    - `removeBySessionId(sessionId)` - 移除映射

### WebSocketService

提供5个业务方法：

1. `broadcastChatMessage(gameId, message)` - 广播聊天消息
2. `notifyGamePlayerScriptReady(gamePlayerId, scriptId)` - 通知剧本就绪
3. `notifyPlayerStartInvestigation(gamePlayerId, scenes)` - 通知开始搜证
4. `broadcastPublicClues(gameId, clues)` - 广播公开线索
5. `broadcastVoteResult(gameId, murdererId, voteDetails, dmScore)` - 广播投票结果

## 扩展指南

### 添加新的消息类型

1. 在 `WebSocketMessage.MessageType` 添加枚举值
2. 在 `WebSocketService` 添加业务方法
3. 在 `WebSocketServiceImpl` 实现发送逻辑

### 添加新的接收端点

1. 在 `WebSocketController` 添加 `@MessageMapping` 方法
2. 路径格式：`/game/{gameId}/xxx`

## 注意事项

1. **线程安全**：PlayerSessionManager 使用 ConcurrentHashMap，已保证线程安全
2. **连接顺序分配**：每个游戏的分配计数器在 GameHandshakeInterceptor 中维护
3. **重连处理**：玩家断开重连时会重新分配 gamePlayerId（可能相同）
4. **无注册接口**：前端不需要显式注册，连接时自动完成分配
